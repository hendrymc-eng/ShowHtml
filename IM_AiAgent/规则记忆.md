## 样式铁律补充（2026-06-15）
- **Agent Center 的 `.ac-mag-stats` 是顶部核心信息模块，任何首屏压缩/配置入口/Hallmark 优化都不得把它裁切、隐藏或压到不可读。**
- 优化 Agent Center 时必须先验证：3 个 stats 全部可见、`statsInsideHero=true`、每个 stat 子项 `scrollHeight <= clientHeight + 1`、无 TabBar 遮挡。
- 若首屏空间不够，优先压缩 `.ac-core-card` 和 `.ac-config-entry`，不要牺牲 `.ac-mag-stats`；原则仍是“缩要素，不删要素”。

- **Agent Center 的 `.ac-core-btn` 是销售主卡核心操作按钮，任何压缩 `.ac-core-card` 时都必须保证按钮完整在卡片内部可见。** 验证条件：按钮 `bottom <= core.bottom - 2`、`top >= core.top + 2`、高度 ≥ 24px；不得用 `overflow:hidden` 把按钮裁掉。
- Agent Center 下方模块（`.ac-scene-section` / `.ac-open-section` / `.ac-library-section`）不能出现“大块空白间隔”。优化时压缩 grid gap、section margin、row padding 和 safe space，而不是只堆 margin 或固定大高度。

- 老板要的是"完整原型模型"——**不是介绍页**，是**可点击的真 app 体验**
- 一个手机模型就是完整原型，点开就能用、能切 tab、能跟 Agent 聊
- 反对：营销落地页、杂志介绍章节、手机外部大段说明、banner + 小预览

## 关键 bug
- JS 修改后必须使用 no-cache server（`python3 serve.sh`）或 cache buster —— 浏览器强缓存旧 JS，函数定义丢失
- 非首页 view 必须自动 `.active`，否则 `.phone-view {display:none}` 导致点击后手机内容空白
- `.phone-screen` 必须 `align-items: stretch`，否则旧 `base.css` 会把 `.phone-content` 宽度压成 0px
> **冲突时本文件优先**（项目特化 > 通用约定）。

## 🤖 3 个 AI Agents 详细分析

### Agent 1：客服 Agent（咨询问题 / 自动解答）— P0

- **目标**：客户咨询问题，自动解答
- **触发**：客户在 IM 中发起咨询
- **入口**：通讯录 → 客服号（已有 UI 位置，**零侵入**）
- **输入**：客户问题（文字/语音/图片）
- **输出**：自动回答 / 转人工 / 留资
- **能力清单**：
  - FAQ 知识库匹配
  - 多轮对话
  - 意图识别
  - 转人工（无缝衔接）
  - 满意度调研
- **知识库**：FAQ + 业务文档 + 历史工单
- **难度**：🟢 低
- **价值**：🎯 立刻降低客服成本

### Agent 2：AI 销冠 Agent（自动与客人沟通 / 引导下单）— P0 ⭐ 核心差异化

- **目标**：自动与客人沟通，**引导下单**（销售转化）
- **触发**：客人进店 / 主动咨询 / 公众号引流 / 广告落地页接入
- **入口**（待设计阶段确认）：
  - 方案 A：通讯录 → 销售号（类似客服号）
  - 方案 B：消息页 → 加号菜单（新增"销售助手"）
  - 方案 C：第三方引流对接（公众号/抖音/小红书）
- **输入**：客人问题/需求/浏览行为
- **输出**：产品推荐 / 议价 / 优惠 / 下单链接 / 跟进
- **能力清单**：
  - 销售话术（开场/挖需/介绍/异议/促单）
  - 产品知识库（SKU/价格/库存）
  - 个性化推荐（基于客人画像/历史）
  - 主动跟进（催付/二次营销）
  - **引导下单**（核心 KPI：转化率）
- **目标客户**：**UmakeX 里面的用户**（老板拍板）
- **与客服 Agent 的区别**：
  - 客服：被动应答 → 解决疑问
  - 销冠：主动引导 → 创造交易
- **难度**：🟡 中-高
- **价值**：🎯 核心差异化，直接带来收入

### Agent 3：会议 Agent（会议纪要 / 文档总结）— P1

- **目标**：会议纪要 + 文档总结
- **触发**：会议开始 / 会议结束 / 文档上传
- **入口**：
  - 会议 tab（实时会议 + 会议回放）
  - 我的 → 应用中心（文档总结入口）
- **输入**：会议录音 / 文档（PDF/Word/PPT）
- **输出**：
  - 会议纪要（要点 + 决议 + 行动项）
  - 文档摘要（关键信息 + 待办）
  - 多语言翻译
- **能力清单**：
  - ASR（语音转文字）
  - 说话人分离
  - 关键信息抽取（人名/时间/任务）
  - 总结生成（结构化/可编辑）
  - 行动项跟踪
- **难度**：🟡 中
- **价值**：🎯 提升会议效率，沉淀知识
