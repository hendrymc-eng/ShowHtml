## 3. 关键用户旅程

### 3.1 客服咨询（Agent 1）
```
客户进 IM
  → 通讯录 tab
  → 找到"客服号"（列表顶部）
  → 点击进入客服对话
  → 输入问题（如"如何退款"）
  → 客服 Agent 自动回复（mock 假回复）
  → 多轮对话（mock 假对话流）
  → 必要时"转人工"按钮
```

### 3.2 销冠引导（Agent 2）⭐
```
客户首次进入
  → 消息页加号菜单（新增"销售助手"）
  或 通讯录 → 销售号（类似客服号）
  → 销冠 Agent 主动问候
  → 客户表达需求（mock 输入）
  → 销冠推荐产品（mock 假产品库）
  → 议价 / 优惠
  → 引导下单（mock 支付按钮）
  → 跟进
```

### 3.3 会议纪要（Agent 3）
```
开会
  → 会议详情页
  → 点击"开始录制"
  → 录音 mock（实际是写死的"录音"提示）
  → 会议结束
  → 自动出纪要（mock 假纪要）
  → 行动项 + 要点
  → 导出 / 分享（mock 按钮）
```

## 4. UI/UX 设计规范

### 4.1 风格定位（核心：借鉴 + 创新）

| 维度 | UmakeX 借鉴 | 老板创新（§4 规则） |
|---|---|---|
| 色调 | 浅色（白+浅蓝+浅灰） | **暖白底 #faf8f5 + 黑字 + 红强调 #ff3b30** |
| 字体 | 系统默认 | **DM Mono（标签） + Noto Serif SC（标题）** |
| 布局 | 卡片式 4 tab | **杂志编辑风**（不对称/留白/分栏） |
| 标签 | 普通 | **DM Mono + 红 kicker 标签** |
| 进度条 | 无 | ❌ **禁 hbar**（用竖立柱/阶梯/金字塔） |
| 展示方式 | 标准手机 | **手机模型展示**（CSS 画 74×148px 手机壳） |
| 动效 | 简单 | **悬停微动效 + 入场动画** |

### 4.4 关键组件规范

- **手机模型**：CSS 画 74×148px 圆角矩形（border-radius: 12px），放页面截图
- **Kicker**：红色小标签 + 大写英文（如"AI AGENT"）
- **对话气泡**：己方 12px 圆角+右下角 0；对方 12px 圆角+左下角 0
- **AI 头像**：圆形 + 渐变背景 + 居中"AI"字母

- ❌ 诗化标题文案（"温柔的力量"类）
- ❌ 一次性弹出所有信息
- ❌ 大段灰色文字墙

## 5. 技术方案

### 5.1 技术栈

| 维度 | 选型 | 理由 |
|---|---|---|
| 前端 | **纯 HTML + CSS + JS** | 单页/多子页，无框架（轻） |
| CSS | **原生 CSS（变量 + Flex/Grid）** | 不引外部库 |
| JS | **原生 ES6** | 不引框架（避免 build step） |
| 图标 | **SVG 内嵌 / Emoji** | 0 依赖 |
| 字体 | **Google Fonts CDN**（DM Mono + Noto Serif SC） | 0 部署 |
| 数据 | **JSON / JS 对象写死** | 不接后端（v4 范围） |
| AI | **❌ 不接 API** | v4 范围：只要 mock |
| 部署 | **本地 file:// 打开** / GitHub Pages | 老板演示用 |

### 5.2 mock AI 实现方案

**前端**（`js/mock-ai.js`）：
```js
// 客服 Agent 假回复模板
const customerReplies = [
  "您好，我是智能客服小 U。请问您要咨询什么？",
  "关于您的问题，建议您查看订单详情...",
  "我已为您查询，订单状态是...",
  "如需转人工，请点击下方按钮",

// 销冠 Agent 假回复模板
const salesReplies = [
  "您好！我是销售小销。看到您对我们的产品感兴趣...",
  "根据您的需求，我推荐...",
  "现在下单享受 8 折优惠，点击这里立即购买",
  "已为您生成订单，付款链接：...",
];

// 会议 Agent 假纪要
const meetingSummary = {
  title: "Q3 营销策略会",
  duration: "60 分钟",
  attendees: ["张三", "李四", "王五"],
  keyPoints: ["...", "...", "..."],
  decisions: ["...", "..."],
  actionItems: [
    { task: "...", owner: "张三", deadline: "2026-06-15" }
  ]

```

**交互逻辑**（前端 JS）：
- 用户输入消息 → 显示在对话区
- 500ms 后触发 mock 回复（从模板选一条）
- 假回复带"AI"角标 + 打字动画
- 销冠特殊：定期推送"产品推荐卡片"
- 会议特殊：点击"开始录制"→ 倒计时 → 出纪要

### 5.3 页面间跳转

- 用 `<a href="pages/xxx.html">` 标准链接
- 部分交互用 JS 切换 `display: none/block`（同页内）
- 不用 SPA 框架

## 8. 风险点 + 应对

| # | 风险 | 等级 | 应对 |
|---|---|---|---|
| 1 | **大工作量**：14 页 + 3 Agents + mock | 🟡 | **分批交付**：第一批核心 6 页 + Agent 中心 → 老板看效果 → 补剩余 |
| 2 | **风格融合冲突**：UmakeX 冷色 vs 老板暖色 | 🟡 | **分层**：IM 主体保持 UmakeX 冷色；AI Agents 展示页用老板暖色 + 杂志风 |
| 3 | **mock 假回复不像真对话** | 🟡 | **写至少 5 轮对话流**，不是单条回复；加打字动画 + AI 角标 |
| 4 | **手机模型展示效果**（老板铁律） | 🟢 | **每页都要有 74×148px 手机模型**（CSS border-radius + 阴影） |
| 5 | **可点击 vs 死链** | 🟡 | **所有 tab/按钮/列表项都要能跳**，跳到 mock 页或显示 mock 弹窗 |
| 6 | **验收标准模糊** | 🟡 | **明确合格线**：每页都有内容、每按钮都能跳、3 Agents 都能输入并出假回复 |
